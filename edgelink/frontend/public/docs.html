Overview

EdgeLink is a developer-first URL shortener built on Cloudflare's edge network. It provides powerful features for link management, analytics, and smart routing.

Base URL

https://go.shortedbro.xyz

Features by Plan

Feature Free Pro
Short Links 25 Unlimited
Monthly Clicks 1,000 100,000
Analytics Basic (7 days) Advanced (30 days)
Custom Slugs Yes Yes
Link Expiration Yes Yes
Device Routing No Yes
Geographic Routing No Yes
A/B Testing No Yes
Custom Domains No 3 domains
Webhooks No Yes
Link Groups No 10 groups

Authentication

EdgeLink supports two authentication methods:

1. JWT Tokens (Recommended for Users)

Short-lived tokens (24 hours) for user sessions. Get a token by signing up or logging in.

Sign Up

POST /auth/signup 
curl -X POST https://go.shortedbro.xyz/auth/signup \
  -H "Content-Type: application/json" \
  -d '{
    "email": "you@example.com",
    "password": "SecurePass123",
    "name": "Your Name"
  }'
Response
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "refresh_token": "rt_abc123...",
  "expires_in": 86400,
  "token_type": "Bearer",
  "user": {
    "user_id": "usr_abc123",
    "email": "you@example.com",
    "plan": "free"
  }
}

Login

POST /auth/login 
curl -X POST https://go.shortedbro.xyz/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "you@example.com",
    "password": "SecurePass123"
  }'

Refresh Token

POST /auth/refresh 
curl -X POST https://go.shortedbro.xyz/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{
    "refresh_token": "rt_abc123..."
  }'

2. API Keys (Recommended for Automation)

Long-lived tokens (1 year) for programmatic access. Generate from the dashboard or API.

Using API Key 
curl -X POST https://go.shortedbro.xyz/api/shorten \
  -H "Authorization: Bearer elk_abc123..." \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com"}'
Note: All API keys start with elk_ prefix. JWT tokens start with eyJ.

Rate Limits

Rate limits are applied per user/IP based on plan:

Plan Limit Period
Anonymous 10 requests Per hour
Free 100 requests Per day
Pro 5,000 requests Per day

Rate Limit Headers

Every response includes rate limit information:

X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1699920000

Rate Limit Exceeded Response

{
  "error": "Rate limit exceeded",
  "code": "RATE_LIMIT_EXCEEDED",
  "retry_after": 3600
}

Error Handling

All errors return a consistent JSON format:

{
  "error": "Human-readable error message",
  "code": "ERROR_CODE",
  "details": { }
}

Common Error Codes

Code Status Description
INVALID_INPUT 400 Request body validation failed
UNAUTHORIZED 401 Missing or invalid authentication
FORBIDDEN 403 Feature requires Pro plan
NOT_FOUND 404 Resource not found
SLUG_EXISTS 409 Custom slug already taken
RATE_LIMIT_EXCEEDED 429 Too many requests
PLAN_LIMIT_EXCEEDED 403 Plan usage limit reached

Create Short Links

Create shortened URLs with optional customization.

POST /api/shorten

Basic Example

cURL 
curl -X POST https://go.shortedbro.xyz/api/shorten \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/very-long-url"
  }'

With Custom Slug

cURL 
curl -X POST https://go.shortedbro.xyz/api/shorten \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/product-launch",
    "custom_slug": "launch2024"
  }'

All Parameters

Parameter Type Required Description
url string Required Destination URL (must be valid URL)
custom_slug string Optional Custom short code (3-50 chars, alphanumeric + hyphens)
custom_domain string Optional Your verified custom domain Pro
expires_at string Optional ISO 8601 expiration date
timezone string Optional IANA timezone (e.g., "Asia/Kolkata")
max_clicks number Optional Expire after N clicks
password string Optional Password protect the link Pro
utm_template string Optional UTM parameters to append
device_routing object Optional Device-based destinations Pro
geo_routing object Optional Country-based destinations Pro

Full Example with All Options

cURL 
curl -X POST https://go.shortedbro.xyz/api/shorten \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/campaign",
    "custom_slug": "summer-sale",
    "custom_domain": "links.yourdomain.com",
    "expires_at": "2024-12-31T23:59:59Z",
    "timezone": "America/New_York",
    "max_clicks": 1000,
    "password": "secret123",
    "utm_template": "?utm_source=social&utm_medium=twitter&utm_campaign=summer"
  }'
Response
{
  "slug": "summer-sale",
  "short_url": "https://links.yourdomain.com/summer-sale",
  "expires_in": 2592000,
  "qr_code_url": "/api/qr/summer-sale"
}

Parameter Type Description

Edit Slug & Destination

Update the destination URL or change the slug of an existing link.

PUT /api/links/{slug}

Update Destination Only

cURL 
curl -X PUT https://go.shortedbro.xyz/api/links/my-link \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "destination": "https://new-destination.com/page"
  }'

Change Slug (Rename Link)

cURL 
curl -X PUT https://go.shortedbro.xyz/api/links/old-slug \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "destination": "https://example.com",
    "new_slug": "new-slug"
  }'

Update Both

cURL 
curl -X PUT https://go.shortedbro.xyz/api/links/old-campaign \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "destination": "https://example.com/new-campaign",
    "new_slug": "campaign-2024"
  }'
Response
{
  "message": "Link updated successfully",
  "slug": "campaign-2024",
  "destination": "https://example.com/new-campaign",
  "short_url": "https://go.shortedbro.xyz/campaign-2024"
}
Important: Changing the slug will break existing links. The old URL will no longer work. Make sure to update any places where the old link is shared.

Device Routing Pro

Route visitors to different destinations based on their device type (mobile, tablet, desktop).

POST /api/links/{slug}/routing/device

Set Device Routing

cURL 
curl -X POST https://go.shortedbro.xyz/api/links/my-app/routing/device \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "mobile": "https://apps.apple.com/app/myapp",
    "tablet": "https://play.google.com/store/apps/details?id=myapp",
    "desktop": "https://myapp.com/download"
  }'

Create Link with Device Routing

cURL 
curl -X POST https://go.shortedbro.xyz/api/shorten \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://myapp.com",
    "custom_slug": "get-app",
    "device_routing": {
      "mobile": "https://apps.apple.com/app/myapp",
      "tablet": "https://play.google.com/store/apps/details?id=myapp",
      "desktop": "https://myapp.com/download"
    }
  }'

Get Current Device Routing

cURL 
curl -X GET https://go.shortedbro.xyz/api/links/my-app/routing \
  -H "Authorization: Bearer YOUR_TOKEN"
Note: The default destination (set in the main URL) is used when no device-specific route matches or when device routing is not configured for that device type.

UTM Parameters

Automatically append UTM tracking parameters to your destination URLs for analytics tracking.

Create Link with UTM Parameters

cURL 
curl -X POST https://go.shortedbro.xyz/api/shorten \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://mysite.com/landing",
    "custom_slug": "twitter-promo",
    "utm_template": "?utm_source=twitter&utm_medium=social&utm_campaign=summer2024&utm_content=banner"
  }'

UTM Parameters Explained

Parameter Purpose Example
utm_source Traffic source google, facebook, twitter, newsletter
utm_medium Marketing medium cpc, social, email, banner
utm_campaign Campaign name summer_sale, product_launch
utm_term Paid keywords running+shoes
utm_content Content variant logo_link, text_link

Example with All UTM Parameters

cURL 
curl -X POST https://go.shortedbro.xyz/api/shorten \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://shop.com/products",
    "custom_slug": "fb-sale",
    "utm_template": "?utm_source=facebook&utm_medium=paid_social&utm_campaign=black_friday_2024&utm_term=shoes&utm_content=carousel_ad_1"
  }'

When someone clicks go.shortedbro.xyz/fb-sale, they'll be redirected to:

https://shop.com/products?utm_source=facebook&utm_medium=paid_social&utm_campaign=black_friday_2024&utm_term=shoes&utm_content=carousel_ad_1

A/B Testing Pro

Split test different destinations to optimize conversion rates.

POST /api/links/{slug}/ab-test

Create A/B Test

cURL 
curl -X POST https://go.shortedbro.xyz/api/links/landing/ab-test \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "variant_a": "https://mysite.com/landing-v1",
    "variant_b": "https://mysite.com/landing-v2"
  }'

Traffic is automatically split 50/50 between variants. Each visitor consistently sees the same variant (using cookies).

Verifying Webhook Signatures

All webhooks include a signature in the X-EdgeLink-Signature header for verification.

Node.js 
const crypto = require('crypto');

function verifyWebhook(payload, signature, secret) {
  const expectedSignature = crypto
    .createHmac('sha256', secret)
    .update(payload)
    .digest('hex');

  return crypto.timingSafeEquals(
    Buffer.from(signature),
    Buffer.from(expectedSignature)
  );
}

// Express middleware
app.post('/webhook', express.raw({type: 'application/json'}), (req, res) => {
  const signature = req.headers['x-edgelink-signature'];
  const payload = req.body.toString();

  if (!verifyWebhook(payload, signature, 'your-webhook-secret')) {
    return res.status(401).send('Invalid signature');
  }

  const event = JSON.parse(payload);
  console.log('Received event:', event.event);

  // Process the webhook...

  res.status(200).send('OK');
});
Python 
import hmac
import hashlib
from flask import Flask, request

app = Flask(__name__)

def verify_webhook(payload, signature, secret):
    expected = hmac.new(
        secret.encode(),
        payload,
        hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(signature, expected)

@app.route('/webhook', methods=['POST'])
def webhook():
    signature = request.headers.get('X-EdgeLink-Signature')
    payload = request.get_data()

    if not verify_webhook(payload, signature, 'your-webhook-secret'):
        return 'Invalid signature', 401

    event = request.get_json()
    print(f"Received event: {event['event']}")

    # Process the webhook...

    return 'OK', 200
